Accessing Hands by Index or ID

 Top  Previous  Next

The hand data is represented by the IHand interface. This includes information about the hand position, posture, joints, etc. You can retrieve a specific hand’s data from the PXC[M]HandData object.

There are two types of identifiers by which you can access hand data:

•Unique hand ID - Each identified hand is assigned a unique numeric ID, which is constant for that hand throughout the session. Unique hand ID values are not guaranteed to be sequential or limited in size. To access the same hand throughout the session, use the hand ID. To retrieve a hand’s unique ID, you can query for it using one of the order-based indexes described below. You may also receive the unique ID values in the AlertData and GestureData structures passed to the callback functions.
•Order-based index – Hands may be indexed according to several types of orders, such as by time (the first hand detected will have a zeor index) or by distance (the hand closest to the sensor will have a zero index). Order-based indexes are sequential and start from zero.

The following table describes the different types of order-based indexes, which are defined in the AccessOrderType enumerator:

Order Type

Description

ACCESS_ORDER_BY_ID

Access the hand according to its unique ID.

ACCESS_ORDER_BY_TIME

Order according to the time that the hand was detected (“oldest” hand has index zero).

ACCESS_ORDER_NEAR_TO_FAR

Order according to the distance of the hand from the sensor (nearest hand has index zero).

ACCESS_ORDER_LEFT_HANDS

Retrieve only left hands. (Usually there is only one left hand in view. If there are more, the order of their retrieval is unpredictable).

ACCESS_ORDER_RIGHT_HANDS

Retrieve only right hands. (Usually there is only one right hand in view. If there are more, the order of their retrieval is unpredictable).

ACCESS_ORDER_FIXED

The first hand to enter the scene has an index of zero, and the second hand to enter the scene has an index of one.

Using the different access order types does not affect the hands that are tracked. The first two hands to enter the scene are always tracked.

Example 64 and Example 65 show two alternative methods for accessing the information of the hand nearest to the sensor.

C++ Example 64: Retrieve the Hand Data by ID

// handData is a PXCHandData instance

// Update hand data

handData->Update();

 

// retrieve the hand identifier

pxcUID handId;

handData->QueryHandId(PXCHandData::ACCESS_ORDER_NEAR_TO_FAR, 0, handId);

// You can keep the unique hand ID for use throughout the session

 

// retrieve the hand data by unique hand ID

PXCHandData::IHand *ihand=0;

handData->QueryHandDataById(handId, ihand);

C# Example 64: Retrieve the Hand Data by ID

// handData is a PXCMHandData instance

// Update hand data

handData.Update();

 

// retrieve the hand identifier

Int32 handId;

handData.QueryHandId(PXCMHandData.AccessOrder.ACCESS_ORDER_NEAR_TO_FAR, 0, out handId);

// You can keep the unique hand ID for use throughout the session

 

// retrieve the hand data by unique hand ID

PXCMHandData.IHand ihand;

handData.QueryHandDataById(handId, out ihand);

Java Example 64: Retrieve the Hand Data by ID

// handData is a PXCMHandData instance

// Update hand data

handData.Update();

 

// retrieve the hand identifier

int[] handId=new int[1];

handData.QueryHandId(PXCMHandData.AccessOrder.ACCESS_ORDER_NEAR_TO_FAR, 0, handId);

// You can keep the unique hand ID for use throughout the session

 

// retrieve the hand data by unique hand ID

PXCMHandData.IHand ihand=new PXCMHandData.IHand();

handData.QueryHandDataById(handId, ihand);
