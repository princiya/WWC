Description

The DeviceInfo structure provides the capture device information.

Members

 

name

The NULL-terminated (C++) user-friendly device name string.

 

serial

The serial number string of the device.

 

did

The NULL-terminated (C++) unique string to identify the capture device, or the symbolic link of the device.

 

firmware

The firmware version of the device.

 

location

The device location relative to be lower left of the display panel, positive X to the right and positive Y to up. The values are in mm.

 

orientation

The device orientation.

 

streams

The bit-OR'ed value of all stream types the device supports.

 

didx

The zero-based device index if there are multiple devices on the system.

 

duid

The unique identifier of the device. The identifier is randomly generated and remains unique for a specific SDK session. Mostly used in power management to identify the device.

Example

Example 130 shows how to enumerate all device information.

C++ Example 130: Enumerate Device Information

// PXCCapture *capture;

for (int i=0;;i++) {

   PXCCapture::DeviceInfo dinfo;

   if (capture->QueryDeviceIfo(i,&dinfo)<PXC_STATUS_NO_ERROR) break;

   wprintf_s(L"device[%d]: %s\n", i, dinfo.name);

}

C# Example 130: Enumerate Device Information

// PXCMCapture capture;

for (int i=0;;i++) {
