Configuring the Touchless Controller Module

 Top  Previous  Next

Before activating the touchless controller module, or at any time during its operation, you can enable or disable its configuration options by calling the SetProfile function (see Integrating with the Module). These options relate to enabling specific UI controls and event injection. The configuration options are set using a single bit-mask variable (the configuration “profile”), where each bit represents a different option. A bit value of 1 enables the option, and a bit value of 0 disables the option.

Table 13 describes the supported configuration options:

Configuration Option

Description

Configuration_None

All touchless control options are disabled.

Configuration_Allow_Zoom

Enable the Zoom action. See Zoom for more details.

Configuration_Use_Draw_Mode

Deprecated.

Configuration_Scroll_Horizontally

Enable horizontal scrolling. See Scroll for more details.

Configuration_Scroll_Vertically

Enable vertical scrolling. See Scroll for more details.

Configuration_Meta_Context_Menu

Enable the meta menu feature. See Activate Meta Menu for more details.

Configuration_Enable_Injection

Enable injection of keyboard and touch events directly to the OS. See Handling Controls with Automatic Injection and Event Handler for more details.

Configuration_Edge_Scroll_Horizontally

Enable horizontal scrolling when the pointer is on the right or left edge of the screen. This is another way to activate scrolling, in addition to the user’s explicit scroll gesture.

Configuration_Edge_Scroll_Vertically

Enable vertical scrolling  when pointer is on the top or bottom edge of the screen. This is another way to activate scrolling, in addition to the user’s explicit scroll gesture.

Configuration_Allow_Back

Enable the Back action. See Back for more details.

Configuration_Allow_Selection

Enable the Select action. See Select for more details.
