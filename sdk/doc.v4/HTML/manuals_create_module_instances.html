Creating Module Instance

 Top  Previous  Next

The simplest way is to create a module instance based on its interface. As illustrated in Example 119, you can create an instance of the capture module by using the CreateImpl function. The SDK searches for the first module that implements the specified interface and creates an instance of the module. This works best if you simply uses the interface functionalities and does not customize the results for a particular module.

C++ Example 119: Create the Module Instance with the Interface Identifier

// session is a PXCSession instance

PXCCapture *capture;

session->CreateImpl<PXCCapture>(&capture);

C# Example 119: Create the Module Instance with the Interface Identifier

// session is a PXCMSession instance

PXCMCapture capture;

session.CreateImpl<PXCMCapture>(out capture);

Java Example 119: Create the Module Instance with the Interface Identifier

// session is a PXCMSession instance

PXCMCapture capture=new PXCMCapture();

session.CreateImpl(capture);

In some cases, you may need to pinpoint a module for customization. This can be done in two ways:

1.If you already knows the module’s implementation identifier, you can call the CreateImpl function with the implementation identifier, as illustrated in Example 120. The interface identifier and the implementation identifier uniquely identify a module implementation. You can find the implementation identifier by calling the QueryImpl function, which enumerates all available modules. See Enumerating Modules for details.

C++ Example 120: Create the Module Instance with the Unique Module Identifier

// session is a PXCSession instance

PXCFaceModule *fd;

session->CreateImpl<PXCFaceModule>(0x43434343&fd);

C# Example 120: Create the Module Instance with the Unique Module Identifier

// Create an algorithm instance

PXCMFace3D face;

session.CreateImpl<PXCMFace3D>(0x43434343, out face);

Java Example 120: Create the Module Instance with the Unique Module Identifier

// Create an algorithm instance

PXCMFaceModule face=new PXCMFaceModule();

session.CreateImpl(0x43434343, face);
2.Alternatively, you can fill out the ImplDesc structure that details different aspects of a module. The CreateImpl function finds a match and creates an instance of the module, as illustrated by Example 121. Here the ImplDesc structure serves as a search template if not completely filled. Zero values in a structure field match any values. A bit-pattern in a bit-OR’ed field matches any bit patterns that are a super set.

C++ Example 121: Create Module Instance with Descriptor

// session is a PXCSession instance

PXCSession::ImplDesc desc;

memset(&desc,0,sizeof(desc));

desc.group=PXCSession::IMPL_GROUP_OBJECT_RECOGNITION;

desc.subgroup=PXCSession::IMPL_SUBGROUP_FACE_ANALYSIS;

desc.vendor=0x8086;        // Intel

 

PXCFaceModule *fd;

session->CreateImpl<PXCFaceModule>(&desc,&fd);

C# Example 121: Create Module Instance with Descriptor

// session is a PXCMSession instance

PXCMSession.ImplDesc desc=new PXCMSession.ImplDesc();

desc.group=PXCMSession.ImplGroup.IMPL_GROUP_OBJECT_RECOGNITION;

desc.subgroup=PXCMSession.ImplSubgroup.IMPL_SUBGROUP_FACE_ANALYSIS;

desc.vendor=0x8086;        // Intel

 

PXCMFaceModule face;

session.CreateImpl<PXCMFaceModule>(desc, out face);

Java Example 121: Create Module Instance with Descriptor

// session is a PXCMSession instance

PXCMSession.ImplDesc desc=new PXCMSession.ImplDesc();

desc.group=PXCMSession.ImplGroup.IMPL_GROUP_OBJECT_RECOGNITION;

desc.subgroup=PXCMSession.ImplSubgroup.IMPL_SUBGROUP_FACE_ANALYSIS;

desc.vendor=0x8086;        // Intel

 

PXCMFaceModule face=new PXCMFaceModule();

session.CreateImpl(desc, face);
