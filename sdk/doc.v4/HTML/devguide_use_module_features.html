Use Module Features

 Top 

The SDK session manages I/O modules and processing modules. Many modules implement the same interface and provide similar feature set. Thus unless the application is designed to the specific behavior of a specific module implementation, it is recommended that the application do not pin point a module implementation. Here pinpointing a module implementation means using the module unique identifier, or other module specific parameters to locate and create the module instance. Instead, the application can create a module instance by specifying the interface identifier, as illustrated in Example 162.

C++ Example 162: Create a Module Instance by Interface Identifier

// session is a PXCHandAnalysis instance.

PXCHandAnalysis *hand;

session->CreateImpl<PXCHandAnalysis>(&hand);

C# Example 162: Create a Module Instance By Interface Identifier

// session is a PXCMHandAnalysis instance.

PXCMHandAnalysis hand;

session.CreateImpl<PXCMHandAnalysis>(PXCMHandAnalysis.CUID,out hand);

To ensure that the required features are supported, it is recommended that the application check the features before using them. The application can check the feature set by calling the corresponding QueryProfile function, as illustrated in Example 163.

C++ Example 163: Check Finger Tracking Feature Set

// hand is a PXCHandAnalysis instance

PXCHandAnalysis::ProfileInfo pinfo;

// Retrieve the working configuration

hand->QueryProfile(&pinfo);

// Check features...

C# Example 163: Check Finger Tracking Feature Set

// hand is a PXCMHandAnalysis instance

PXCMHandAnalysis.ProfileInfo pinfo;

// Retrieve the working configuration

hand.QueryProfile(out pinfo);

// Check features...
