Configuring the Blob Module

 Top  Previous  Next

The blob module has various configuration options, such as the number and size of objects that can be identified and output as blobs, etc. You can configure the blob module with the PXC[M]BlobConfiguration interface. The code sample below demonstrates how to do this.

Call the ApplyChanges function to apply any new configuration.

C++ Example 56: Configure the Blob Module

// Get an instance of PXCBlobConfiguration

PXCBlobConfiguration* blobConfig = blobModule->CreateActiveConfiguration();

 

// Configure the blob module

blobConfig->SetSegmentationSmoothing(0.7);

// … call more configuration methods

 

// Apply the new configuration values

blobConfig->ApplyChanges();

C# Example 56: Configure the Blob Module

// Get an instance of PXCMBlobConfiguration

PXCMBlobConfiguration blobConfig = blobModule.CreateActiveConfiguration();

 

// Configure the blob module

blobConfig.SetSegmentationSmoothing(0.7f);

// … call more configuration methods

 

// Apply the new configuration values

blobConfig.ApplyChanges();

Java Example 56: Configure the Blob Module

// Get an instance of PXCMBlobConfiguration

PXCMBlobConfiguration blobConfig = blobModule.CreateActiveConfiguration();

 

// Configure the blob module

blobConfig.SetSegmentationSmoothing(0.7f);

// … call more configuration methods

 

// Apply the new configuration values

blobConfig.ApplyChanges();

Table 7 summarizes the blob configuration options:

Configuration Function

Description

Default

ApplyChanges

Activates the current configuration on the module. No changes is applied until this function is called.

NA

Update

Retrieves the active configuration.

NA

RestoreDefaults

Retrieves the default configuration.

NA

EnableSegmentationImage

Enable/disable the extraction of the segmentation image.

True

EnableContourExtraction

Enable/disable the extraction of contour lines.

True

SetMaxBlobs

Set the maximal number of blobs to be detected (must be less than 4, the default is 1).

If there are more than the configured number of blobs in the image, the largest objects in the tracking range are extracted.

1

SetMaxDistance

Set the maximal distance in millimeters from the sensor, where objects (blobs) are to be identified.

550 mm

SetMinPixelCount

Set the minimal number of pixels that a blob must have. Objects smaller than this limit are not identified as blobs. The limit must be no more than a quarter of the entire sensor’s image size.

400 pixels

SetMaxObjectDepth

Set the maximal object depth. The object depth is the distance in millimeters between the two object points that are closest and farthest from the sensor. If the object depth is larger than this value, the farthest part of the object is “trimmed”. This feature can be useful if you want to identify only a small part of an object.

300 mm

SetMinContourSize

Set the minimal contour size (in number of contour points) for an object. Any blob whose contour points are fewer than the limit will not be identified.

30 points

SetSegmentationSmoothing

Set the smoothing value for the segmentation image. “Smoothing” will result in a more consistent, continuous image with less “holes”. A higher smoothing value requires more computation resources.

0.3

SetContourSmoothing

Set the smoothing value for the contour line. “Smoothing” causes the contour line to be smoother, i.e. to ignore small inconsistencies in the line. A Higher smoothing value requires more computation resources.

Under extreme conditions, high smoothing values may “split” a single blob in two different outer contour lines.

1

EnableStabilizer

Enable/disable the stabilization of the extremity points. Stabilization may induce a small offset between the extremity points and the contour line.

True
