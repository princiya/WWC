Configuring Object Tracking

Use the following functions to configure the tracking algorithm:

•2D Tracking: Use the Set2DTrackFromFile function to set a reference image from an image file, or the Set2DTrackFromImage function to set a reference image from an image instance. The functions return the target identifier for later retrieval of the tracking data.
•3D Feature Tracking: Use the Set3DTrack function to load the 3DMap (.slam) file generated by the toolbox. The function returns a target identifier for retrieval of the tracking data.
•3D Edge-Based Tracking: Use the Set3DTrack function to set the configuration file (.xml) generated by the toolbox.  The function returns multiple target IDs based on the definition of the XML file.  Typically the first target ID is named TrackingPose and may be used to track the actual position of the tracked object.  The second ID is named InitialPose and may be used to help with aligning the object for initial detection.  As soon as the object is detected, InitialPose no longer reports as tracking. TrackingPose should be used instead.
•3D Instant Tracking (Simultaneous Localization And Mapping or SLAM): Use the Set3DInstantTrack function to enable instant tracking functionality.  In this mode, the tracked object model does not need to be pre-registered with a configuration file.

You can use multiple 2D or 3D feature based tracking models but do not mix 2D and 3D tracking modes. In addition, the edge-based tracking supports only a single model.

If camera calibration data is available, you can use the SetCameraParameters function to set the calibration data.

Tracking Data

During tracking, the PXC[M]Tracker interface provides functions to return the number of tracked objects (see the QueryNumberTrackingValues function), the tracking parameters of the specified object (see the QueryTrackingValues function), and to retrieve all available tracking data (see the QueryAllTrackingValues function).

The tracking data includes the unique object identifier, translation and rotation matrix, as well as other auxiliary information such as time stamp of tracking. See the TrackingValues structure for the detailed definitions.
