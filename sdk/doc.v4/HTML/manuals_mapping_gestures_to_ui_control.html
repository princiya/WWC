Mapping Gestures to UI Controls

 Top  Previous  Next

You may wish to enlarge the list of gestures that trigger touchless UI controls. In addition to the basic UI controls described in UI Controls, you can enhance your application to support touchless controls for moving between items or pages, controlling audio playback and volume, etc.

You can do this by calling the AddGestureActionMapping function, to map certain gestures to standard UI events, optionally supplying a custom handler to handle these events. See Handling Gestures for a list of gestures that can be mapped to UI controls.

You can choose to map one or more additional gestures to one of the following system UI controls:

•Action_LeftKeyPress – left arrow key press.
•Action_RightKeyPress – right arrow key press.
•Action_BackKeyPress – back key press.
•Action_PgUpKeyPress – page up key press.
•Action_PgDnKeyPress – page down key press.
•Action_VolumeUp – volume up Windows* Media* key press.
•Action_VolumeDown – volume down Windows Media key press.
•Action_Mute – mute volume Windows Media key press.
•Action_NextTrack – next track Windows Media key press.
•Action_PrevTrac – previous track Windows Media key press.
•Action_PlayPause – play or pause Windows Media key press.
•Action_Stop – stop playback Windows Media key press.
•Action_ToggleTabs - Windows key + Z key press (used to display the tab menu in Internet Explorer).

You can choose one of two ways to handle the additional gestures:

•Implement and register a custom event handler to handle the occurrence of each additional gesture.
•Don’t supply event handlers to the mapping function. In this case, the standard OS mechanism handles the event to which the gesture is mapped.

Example 88 shows how to map additional gestures to UI controls, without supplying a custom event handler:

C++ Example 88: Map Gestures to UI Controls

touchlessController->AddGestureActionMapping(L“swipeLeft”, PXCTouchlessController::Action_LeftKeyPress);

touchlessController->AddGestureActionMapping(L“swipeRight”, PXCTouchlessController::Action_RightKeyPress);

C# Example 88: Map Gestures to UI Controls

touchlessController.AddGestureActionMapping(“swipeLeft”, PXCMTouchlessController.Action.Action_LeftKeyPress);

touchlessController.AddGestureActionMapping(“swipeRight”, PXCMTouchlessController.Action.Action_RightKeyPress);

Java Example 88: Map Gestures to UI Controls

touchlessController.AddGestureActionMapping(“swipeLeft”, PXCMTouchlessController.Action.Action_LeftKeyPress);

Example 89 shows how to map additional gestures to UI controls, while supplying a custom event handler:

C++ Example 89: Map Gestures to UI Controls with Event Handler

class MyActionHandler : public PXCTouchlessController::ActionHandler {

public:

   virtual void PXCAPI OnFiredAction(const Action action) {

       switch(action) {

       case Action_LeftKeyPress:

           // handle left key press action

           …

           break;

       // handle other actions

       …

   }

};

…

MyActionHandler actionHandler;

touchlessController->AddGestureActionMapping(L“swipeLeft”, PXCTouchlessController::Action_LeftKeyPress, &actionHandler);

touchlessController->AddGestureActionMapping(L“swipeRight”, PXCTouchlessController::Action_RightKeyPress, &actionHandler);

Java Example 89: Map Gestures to UI Controls with Event Handler

class MyActionHandler implements PXCMTouchlessController.ActionHandler {

   public void OnFiredAction(PXCMTouchlessController.Action action) {

       switch(action) {

       case PXCMTouchlessController.Action.Action_LeftKeyPress:

           // handle left key press action

           …

           break;

       // handle other actions

       …

   }

};

 

MyActionHandler handler=new MyActionHandler();

touchlessController.AddGestureActionMapping(“swipeLeft”, PXCMTouchlessController.Action.Action_LeftKeyPress, handler);

touchlessController.AddGestureActionMapping(“swipeRight”, PXCMTouchlessController.Action.Action_RightKeyPress, handler);
