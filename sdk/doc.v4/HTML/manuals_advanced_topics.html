The PXC[M]SenseManager interface provides an easy way of organizing and controlling a multimodal pipeline. However, the interface has the following limitations:

•The pipeline works with a single I/O device.

It is possible to create multiple pipelines to work around this limitation.

•The data flow from the I/O device to the algorithm modules is hard-wired.
•The algorithm control during streaming is limited to pausing/resuming data processing on the algorithm modules.

If the PXC[M]SenseManager interface does not meet your needs, the SDK provides additional interfaces that allow your application to fully control the data and execution flow, and unlock the potential of each algorithm module.
