Handling Controls with Automatic Injection and Event Handler

 Top  Previous  Next

When one of the UI control events occurs, your application must respond accordingly. For instance, if the user triggered a scrolling operation, your application must scroll its display.

Most of the UI control events (all except for Cursor Navigation) can be handled in one of two ways:

•Injection – the module automatically “translates” the controls into standard system touch events (for Select, Scroll and Zoom) or keyboard events (for Back and Go to Start Screen). This means that you do not have to contain special code to handle these events, but rather responds to standard system events. To work in this mode, set the Configuration_Enable_Injection flag in the configuration profile (see Configuring the Touchless Controller Module for more details).
•Event handler – You implement and register an event handler object. When an enabled event fires, the event handler’s callback function is called. The callback function must handle all fired events. See Code Examples for an example of how to implement and register an event handler.

If you need to handle Cursor Navigation events, you must implement an event handler for these events.
