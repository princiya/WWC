The SP, to be returned.

Description

The ReadModuleStreamsAsync function reads the set of image samples from the I/O device for the specified processing module.

This is an asynchronous function. The image samples are invalid until the SP is signaled.

If the processing module requests more than a single stream, the function synchronizes the stream reads so that they are time aligned.

The function maps video streams at the I/O device to the specified processing module, before reading samples. For example, if the I/O device serves a color stream and a depth stream, in that order, and the processing module needs a depth stream only, the function reads the depth stream only from the I/O device and serves to the processing module.

When there are multiple processing modules, the function buffers the reads to each stream so that each processing module will get a chance to read the samples without skipping any samples.

If mid=0, the function reads directly from the I/O device.

Return Status

 

PXC[M]_STATUS_NO_ERROR

The function returned successfully.

 

PXC[M]_STATUS_ITEM_UNAVAILABLE

The I/O device has reached end of file.

 

PXC[M]_STATUS_DEVICE_LOST

The I/O device has been disconnected.

Remarks

Unity Web

This function is not supported in Unity Web. Use the PXCMSenseManager interface for capturing video samples. See the Raw Stream Capturing and Processing section for details.
