Handling Alerts

 Top  Previous  Next

Alerts notify your application of significant events in real-time. These events are occurrences that could affect your application’s behavior, such as hands entering or leaving the scene.

See Configuring Alerts to learn how to enable alerts.

The following table summarizes the types of alerts that may be fired:

Alert Type

Description

ALERT_HAND_DETECTED

A hand is identified and its mask is available.

ALERT_HAND_NOT_DETECTED

A previously detected hand is lost, either because it left the field of view or because it is occluded (hidden by another object).

ALERT_HAND_TRACKED

Full tracking information is available for a hand.

ALERT_HAND_NOT_TRACKED

Tracking information is no longer available for a hand (none of the joints are tracked).

ALERT_HAND_CALIBRATED

Hand measurements are ready and accurate.

ALERT_HAND_NOT_CALIBRATED

Hand measurements are not yet finalized, and are not fully accurate.

ALERT_HAND_OUT_OF_BORDERS

Hand is outside of the tracking boundaries.

ALERT_HAND_INSIDE_BORDERS

Hand has moved back inside the tracking boundaries.      

ALERT_HAND_OUT_OF_LEFT_BORDER

The tracked object is touching the left border of the field of view.

ALERT_HAND_OUT_OF_RIGHT_BORDER

The tracked object is touching the right border of the field of view.

ALERT_HAND_OUT_OF_TOP_BORDER

The tracked object is touching the upper border of the field of view.

ALERT_HAND_OUT_OF_BOTTOM_BORDER

The tracked object is touching the lower border of the field of view.

ALERT_HAND_TOO_FAR

The tracked object is too far to be tracked.

ALERT_HAND_TOO_CLOSE

The tracked object is too close to be tracked.

You can either poll for alerts in your main processing loop, or register alert handlers whose callback functions are called when alerts are fired.

Polling for Alerts within the Main Processing Loop

There are different ways to query for fired alerts within the main processing loop:

•Call IsAlertFired for a specific alert.
•Call IsAlertFiredByHand for a specific alert and a specific hand.
•Call QueryFiredAlertsNumber, and then iterate over the fired alerts by calling QueryFiredAlertData with indexes from 0 to QueryFiredAlertsNumber()-1.

Example 66 illustrates how to poll for specific alerts within the main processing loop.

C++ Example 66: Poll for Specific Alerts

// handData is the PXCHandData instance

 

// poll for alerts

PXCHandData::AlertData data;

if (handData->IsAlertFired(PXCHandData::ALERT_HAND_TRACKED,&data)) {

   // handle HAND_TRACKED alert

   ...

}

 

if (handData->IsAlertFired(PXCHandData::ALERT_HAND_TOO_FAR,&data)) {

   // handle HAND_ TOO_FAR alert

   ...

}

Creating and Registering an Alert Handler

Example 67 illustrates how to create and register an alert handler.

C++ Example 67: Subscribe to Alerts

class MyHandler:public PXCHandConfiguration::AlertHandler {

public:

 

   virtual void PXCAPI OnFiredAlert(const PXCHandData::AlertData &data) {

       // handle alerts

       if (data.label == PXCHandData::ALERT_HAND_DETECTED) {

           // handle hand detected

           …

       } else if (data.label == PXCHandData::ALERT_HAND_TRACKED) {

           …

       }

   };

};

 

// handModule is a PXCHandModule instance

// Subscribe to the alert notification event

PXCHandConfiguration *handConfig=handModule->CreateActiveConfiguration();

MyHandler handler;

handConfig->SubscribeAlert(&handler);

handConfig->ApplyChanges();

 

// streaming & receiving alerts

...

 

// Clean up

handConfig->UnsubscribeAlert(&handler);

handConfig->Release();
