Certain SDK interfaces require special data handling, such as PXC[M]SpeechRecognition and PXC[M]SpeechSynthesis (see Speech Recognition and Speech Synthesis respectively.) Most others follow a generic pattern (see PXC[M]VideoModule) to configure the module and work with the input device (the I/O module).

The Configuration Interface

You can derive the PXC[M]VideoModule instance from the module instance. Example 122 illustrates how to get the PXC[M]VideoModule instance from the face module.

C++ Example 122: Retrieve the Configuration Instance

// face is a PXCFaceModule instance

PXCVideoModule *vm=face->QueryInstance<PXCVideoModule>();

C# Example 122: Retrieve the Configuration Instance

// face is a PXCMFaceModule instance

PXCMVideoModule vm=face.QueryInstance<PXCMVideoModule>();

Java Example 121: Retrieve the Configuration Instance

// face is a PXCMFaceModule instance

PXCMVideoModule vm=(PXCMVideoModule)face.QueryInstance(PXCMVideoModule.CUID);

The Configuration Process

The SDK module configuration is a process that the module negotiates the data needs with the input device. Do the following steps:

•Use the QueryCaptureProfile function to retrieve available profiles. Each profile (the DataDesc structure) describes a set of input data needs that the module can work on. The information includes:
oData Stream Configuration: The module may specify the resolution and/or frame rates directly or as a range of parameters. You need to configure the input device to meet the stream requirements. If the module specifies more than a stream, you need to synchronize the streams and deliver them together to the module.
oDevice Properties and Device Information: The module may tie to a specific device (or a type of devices.) For example, certain modules may work only with user-facing cameras. You need to ensure that the input device is a user-facing camera.
•Locate an input device and configure it to meet the data requests. See Connecting I/O via PXC[M]CaptureManager.
•Use the SetCaptureProfile function to configure the module. You need to fill the DataDesc structure with the actual data specification, for example, the actual stream resolution, frame rate, the device property values and the device information.

After configuration, you can use the module member functions to access the module features.
