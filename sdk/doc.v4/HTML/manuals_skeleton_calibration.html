Skeleton Calibration

 Top  Previous  Next

When a new hand comes into view, the hand module performs automatic calibration to learn its specific measurements, i.e. the distances between its joints. Calibration continues to be refined over time. Tracking begins as soon as a hand is detected, but its accuracy improves significantly when calibration is completed. You can determine when this happens by calling the IsCalibrated function, or by listening for the ALERT_HAND_CALIBRATED event.

If possible, when your application is initializing, it is recommended to prompt the user to make a spread-fingered gesture (such as a tap), to facilitate hand calibration.

If your application provides the hand module with the user name (by calling the SetUserName function), it can save hand calibrations and reload them for each user. This guarantees the best tracking results and may save some time during the initialization process.

It is not mandatory to set the user name, the hand module calibrates correctly regardless of the user name value.

If you can’t save specific calibration data per user, but you know the approximate age of your players (either by prompting the user for his/her age or due to the nature of the application), use the SetDefaultAge function to load the default calibration data suited to the expected age group.
