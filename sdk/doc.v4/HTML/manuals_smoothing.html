Smoothing

 Top  Previous  Next

While stabilization produces more consistent data for a stationary hand, smoothing does the same for a hand in motion. It ignores small deviations in hand motion and overcomes local problems in tracking, to produce smoother, more continuous tracking information. So, for example, the hand’s motion might be represented as a smooth arc, rather than having zigzags in its path.

Unlike stabilization, which is configured as a boolean value, smoothing is configured by degree, as represented by a value between 0 and 1. A value of 0 means no smoothing, while a value of 1 (the default and recommended value) produces the maximal smoothing.

See SetSmoothingValue.

Configuring a smaller smoothing value results in more “jittery” motion but faster reaction time, while a high smoothing value produces smoother motion but introduces some delay.
