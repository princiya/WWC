About Confidence Level

 Top  Previous  Next

The SDK voice module uses confidence level to indicate the likelihood that a returned recognition result is the correct speech.

In general, the application should not rely on the absolute value of the confidence level, as the definition is speech engine specific. It is recommended to use confidence level only in the command and control mode to distinguish among multiple candidates. The higher the associated confidence level, the more likely that the candidate is the correct speech.

However, in some cases, instead of hard-coding a fixed threshold, you can use thresholding to increase or decrease certain aspect of voice recognition. For example, your application may expose a graphical user interface control to let the user adjust what is the acceptable recognition rate. The application can use 50% as the baseline. Any recognition results with a bigger confidence level are accepted and smaller are rejected.  A higher confidence threshold increases false reject or false positive (right command is recognized but rejected) while a lower confidence threshold results in more false accept or false negative (wrong command is recognized and accepted).
