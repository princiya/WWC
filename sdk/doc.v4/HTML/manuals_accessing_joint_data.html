Accessing Joint Data

 Top  Previous  Next

If your application requires specific joint location or speed data, you can use the functions described below to retrieve it.

Joint data is available only in full-hand tracking mode.

The IHand object contains the following methods for retrieving joint-related data:

•IsCalibrated - returns true if there is a valid hand calibration, otherwise returns false. After identifying a new hand, the hand module calculates its calibration. When calibration is complete, an alert is issued. A valid calibration results in more accurate tracking data, that is better fitted to the user's hand.
•HasTrackedJoints - returns true if tracked joint data exists, otherwise returns false. This value is true if joint tracking is enabled and a skeleton was matched to the hand.
•QueryTrackedJoint - retrieve a specific joint’s tracking data.
•HasNormalizedJoints - returns true if normalized joint data exists, otherwise returns false. This value is true if joint normalization is enabled and a skeleton was matched to the hand.
•QueryNormalizedJoint - returns the tracking data of a single joint of the normalized skeleton.
