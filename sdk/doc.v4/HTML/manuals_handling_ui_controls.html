Initializing and Calibrating Hand Tracking

Touchless control is derived from tracked hand motions. Therefore, to begin its operation, the module must identify the user’s hand. To initiate hand tracking, the user must place his or her hand in front of the computer screen, in an open but relaxed posture, as illustrated in Figure 51.

Figure 51: Open Posture to Initiate Tracking

The user can perform UI control gestures with either hand, left or right. Only one hand can control the application at any given time.

The module uses the initial hand placement to make assumptions about the real-world range where the hand moves to control the application. In effect, the module “draws” a virtual 3D cube, with the tracked hand at its center. The hand’s movements in relation to this cube are translated into cursor movements in relation to the screen. For instance, if the hand approaches the right-hand plane of the cube, the cursor moves to the right edge of the screen. The cube’s X, Y and Z dimensions are represented as 1 values, and cursor locations provided in event data are given as (X,Y,Z) values from 0 to 1. (See Event Data for more details).

Every time the hand leaves and returns to the sensor’s field-of-view, its surrounding 3D cube is recalculated.
