Skeleton Normalization

 Top  Previous  Next

Different users have hands of different shapes and sizes. When a new hand comes into view, the Hand Module performs automatic calibration, in order to learn its specific measurements (see Skeleton Calibration). A normalized skeleton is a skeleton of a fixed size and fixed distances between joints. If the normalization feature is enabled, the Hand Module “translates” all hand skeletons to the normalized skeleton. You may prefer to work with a normalized skeleton in the following cases:

•Your application performs hand animation. You may want all users’ hands to be displayed in the same way, regardless of the specific user’s dimensions.
•You are writing your own hand gesture recognition. It’s easier to perform gesture recognition on a normalized skeleton, thus avoiding specific handling of different hand sizes.

It is recommended to disable skeleton normalization if these or similar cases do not apply, in order to save computation resources.
